# 部署计划 — maintenance-decision-service

## 一、概要

将运维决策后端服务（Spring Boot 2.7 + Java 8）以 **Docker 容器** 方式部署到 **Linux (CentOS)** 服务器，用户通过 `http://<服务器IP>:8080` 直接访问。

## 二、当前状态分析

| 项目 | 现状 |
|------|------|
| 构建方式 | Maven (mvnw)，Spring Boot 打包为 fat JAR |
| 数据源 | `src/main/resources/datasets/` 下的 JSON 文件（无需数据库） |
| 端口 | 8080（硬编码于 application.yml） |
| 健康检查 | `/actuator/health` 可用 |
| 文档地址 | `/swagger-ui.html` 和 `/v3/api-docs` |
| 现有部署配置 | 无 Dockerfile、无 docker-compose、无部署脚本 |

## 三、前置条件（服务器准备）

**目标服务器（CentOS）上需要先安装以下软件：**

### 1. 安装 Docker

```bash
# 卸载旧版本（如有）
sudo yum remove -y docker docker-client docker-client-latest docker-common docker-latest docker-latest-logrotate docker-logrotate docker-engine

# 安装 yum-utils 和设置仓库
sudo yum install -y yum-utils
sudo yum-config-manager --add-repo https://download.docker.com/linux/centos/docker-ce.repo

# 安装 Docker Engine
sudo yum install -y docker-ce docker-ce-cli containerd.io docker-buildx-plugin

# 启动 Docker 并设置开机自启
sudo systemctl start docker
sudo systemctl enable docker

# 验证安装
sudo docker --version
```

### 2. 安装 Docker Compose

```bash
# 下载最新版 Docker Compose
sudo curl -L "https://github.com/docker/compose/releases/latest/download/docker-compose-$(uname -s)-$(uname -m)" -o /usr/local/bin/docker-compose

# 赋予执行权限
sudo chmod +x /usr/local/bin/docker-compose

# 验证安装
docker-compose --version
```

### 3. 配置防火墙（放行 8080 端口）

```bash
# 如果使用 firewalld
sudo firewall-cmd --add-port=8080/tcp --permanent
sudo firewall-cmd --reload

# 验证
sudo firewall-cmd --list-ports
```

完成后即可继续后续的部署步骤。

## 四、变更内容

### 1. 新增 `.dockerignore`

**路径**：`/.dockerignore`

**作用**：将无关文件排除在 Docker 构建上下文之外，减小镜像体积。

**内容**：
- 排除 `target/`（本地构建产物）
- 排除 `.mvn/repository/`（本地 Maven 依赖缓存）
- 排除 `.idea/`、`codex-skill/`
- 排除 `*.md`、`mvnw`、`mvnw.cmd`

### 2. 新增 `Dockerfile`

**路径**：`/Dockerfile`

**作用**：多阶段构建，第一阶段用 Maven 编译打包，第二阶段用精简 JRE 运行。

**设计要点**：
- **Builder 阶段**：使用 `maven:3.8-openjdk-8-slim`，复制 `pom.xml` 先下载依赖（利用 Docker 缓存层），再复制源码编译
- **Runtime 阶段**：使用 `eclipse-temurin:8-jre-alpine`（最小化 JRE 镜像，约 80MB），只复制构建好的 JAR
- **健康检查**：添加 `HEALTHCHECK` 指令，定期检查 `/actuator/health`
- **用户安全**：以非 root 用户运行

### 3. 新增 `docker-compose.yml`

**路径**：`/docker-compose.yml`

**作用**：提供一键部署能力，同时方便配置环境变量、端口映射、重启策略和日志限制。

**设计要点**：
- `restart: unless-stopped` —— 服务器重启后自动拉起
- `ports: "8080:8080"` —— 映射宿主 8080 端口
- `logging` 配置 —— 限制日志文件大小，防止磁盘写满
- `healthcheck` —— 覆盖容器级健康检测

### 4. 新增 `deploy.sh`

**路径**：`/deploy.sh`

**作用**：提供一键打包和部署脚本，降低运维人员的操作门槛。

**设计要点**：
- 支持 `./deploy.sh build` —— 仅构建镜像
- 支持 `./deploy.sh start` —— 启动容器
- 支持 `./deploy.sh stop` —— 停止容器
- 支持 `./deploy.sh restart` —— 重启容器
- 支持 `./deploy.sh logs` —— 查看日志
- 支持 `./deploy.sh status` —— 查看状态

### 5. 新增 `application-prod.yml`（可选）

**路径**：`/src/main/resources/application-prod.yml`

**作用**：生产环境配置覆盖，与本地开发配置分离。

**变更点**：
- 可配置日志级别（如 `root: WARN` 减少容器日志量）
- 可配置 `spring.profiles.active=prod` 在 docker-compose 中激活

## 五、假设与决策

| 决策 | 说明 |
|------|------|
| 使用 Docker 多阶段构建 | 无需服务器安装 Maven/JDK，镜像自包含构建环境 |
| 基镜像选择 Temurin 8 JRE Alpine | 体积小（~80MB），安全性高 |
| 直接 IP:8080 访问 | 不引入 Nginx，减少运维复杂度 |
| 非 root 用户运行 | 遵循容器安全最佳实践 |
| 使用 docker-compose 管理 | 比裸 docker run 更易维护和文档化 |
| 数据无持久化需求 | JSON 文件打包在镜像内，不依赖外部卷 |

## 六、验证步骤

1. **服务器 Docker 环境检查**
   ```bash
   sudo docker --version    # 应输出 Docker 版本号
   docker-compose --version  # 应输出 Docker Compose 版本号
   sudo systemctl status docker  # 确认 running
   ```

2. **本地构建验证**
   ```bash
   docker build -t maintenance-decision-service .
   ```
   确认构建成功，无错误

3. **本地启动验证**
   ```bash
   docker run -d -p 8080:8080 --name ops-service maintenance-decision-service
   ```
   访问 `http://localhost:8080/actuator/health` 返回 `{"status":"UP"}`

4. **API 功能验证**
   ```bash
   curl http://localhost:8080/api/v1/aircraft/B-1234/workflow/clarify -X POST -H "Content-Type: application/json" -d "{}"
   ```
   确认返回正常 JSON 响应

5. **Swagger 文档验证**
   浏览器打开 `http://<服务器IP>:8080/swagger-ui.html`
   确认文档正常渲染

6. **服务器部署验证**
   - 传输项目到服务器（git clone 或 scp）
   - 执行 `./deploy.sh build` + `./deploy.sh start`
   - 确认健康检查通过
   - 确认宿主机防火墙 8080 端口已放行
